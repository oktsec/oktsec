## Executive Summary

Oktsec is a **production-ready** security proxy for AI agent-to-agent communication. Single Go binary, no external dependencies, no LLM. Provides Ed25519 identity verification, 159-rule content scanning, policy enforcement, audit trail, and a 7-page real-time dashboard.

**Codebase:** 20,790 LOC Go | 306 tests | 24 test files | 9 packages with >80% test coverage

## Architecture

```
cmd/oktsec/commands/   CLI (19 commands, Cobra)
internal/proxy/        HTTP handler pipeline, stdio proxy, forward proxy, webhooks
internal/dashboard/    Web UI (7 pages, HTMX, SSE live updates)
internal/engine/       Aguara integration (159 detection rules)
internal/audit/        SQLite audit trail (async batch writer, WAL mode)
internal/auditcheck/   Deployment security auditor (41 checks)
internal/config/       YAML config loader + validator
internal/identity/     Ed25519 crypto (sign, verify, keystore)
internal/policy/       ACL evaluator (per-agent can_message)
internal/graph/        Communication topology (betweenness centrality)
internal/discover/     MCP/OpenClaw/NanoClaw auto-detection
internal/mcp/          MCP server mode (6 tools for AI agents)
sdk/                   Go client library
rules/                 Embedded YAML rules (IAP + OpenClaw)
```

### Pipeline Order

```
rate limit -> identity -> suspension -> ACL -> content scan ->
RULE OVERRIDES -> blocked-content -> split injection ->
concatenated scan -> RULE OVERRIDES -> history escalation -> verdict
```

## What's NOT Done (Known Gaps)

### Minor

| Gap | Priority | Notes |
|-----|----------|-------|
| CLI test coverage | Low | keygen, proxy, wrap/unwrap lack unit tests |
| Custom rule schema validation | Low | No runtime validation for user-authored YAML |
| Benchmark harness | Low | cmd/bench exists but not in CI |

### By Design (Out of Scope for v1)

| Limitation | Rationale |
|------------|-----------|
| No Rego/CEL policy language | YAML overrides cover 95% of use cases; complexity not justified |
| No multi-tenant isolation | Single-instance design; orgs use separate instances |
| No multi-step approval workflows | Single reviewer sufficient for current scale |
| Forward proxy limited to Docker Sandbox | General transparent proxy not in scope |
| Rule categories not user-configurable | Categories come from Aguara; custom rules go in custom categories |

## Roadmap Candidates (Not Committed)

These are potential next steps, not planned work:

1. **Webhook retry with backoff** — Currently fire-and-forget; add retry queue for reliability
2. **Rule analytics** — Which rules fire most, false positive rates, trend charts
3. **Agent groups/roles** — Group agents by team/function, apply policies to groups
4. **Prometheus metrics** — /metrics endpoint for Grafana integration
5. **API key auth** — Alternative to Ed25519 for simpler integrations
6. **Multi-instance sync** — Shared audit trail across multiple oktsec instances
7. **Policy diff/history** — Track config changes over time, rollback support
8. **Slack bot** — Interactive quarantine review from Slack (approve/reject buttons)
